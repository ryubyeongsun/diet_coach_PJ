# Diet Coach 프로젝트 개발 진척 상황 보고

- **보고 일자**: 2025년 12월 3일

- **보고자**: Dev A, Dev B

---

## 1. 프로젝트 개요

사용자의 신체 정보와 생활 패턴을 기반으로 TDEE(총 일일 에너지 소비량)를 계산하고, 이를 바탕으로 다이어트 목표 설정, 식단 추천, 관련 상품 쇼핑 기능을 제공하는 웹 애플리케이션 개발을 목표로 합니다.

---

## 2. 현재까지 완료된 사항 (Completed)

### 2.1. 프로젝트 기반 설계 및 환경 구축
- **기술 스택 확립**: `Java 17`, `Spring Boot`, `Maven` 기반의 프로젝트 구조를 확립했습니다.
- **데이터베이스 설계**: `users` 테이블을 중심으로 핵심 데이터 모델의 DB 스키마 설계를 완료했으며, `schema.sql` 파일로 관리하고 있습니다.
- **개발 환경 구축**: `MySQL` 및 `MyBatis` 연동 설정을 완료했으며, 신규 참여자가 즉시 로컬 환경에서 프로젝트를 실행할 수 있도록 상세한 **개발 가이드 문서를 작성**했습니다.

### 2.2. 핵심 아키텍처 및 공통 모듈 구현
- **MVC 아키텍처 적용**: `Controller` - `Service` - `Mapper` 계층으로 역할을 명확히 분리한 **계층형 아키텍처(Layered Architecture)를 적용**했습니다.
- **공통 모듈 구현**:
    - `ApiResponse`: 모든 API의 응답 형식을 통일하기 위한 공통 래퍼 클래스를 구현했습니다.
    - `TdeeCalculator`: BMR, TDEE, 목표 칼로리를 계산하는 핵심 비즈니스 로직을 별도의 공통 모듈로 구현했습니다.
    - `GlobalExceptionHandler`: 프로젝트 전반의 예외를 일관되게 처리하기 위한 전역 예외 핸들러를 구현했습니다.

### 2.3. API 설계 및 기능 구현 현황
- **User 도메인 (구현 완료)**
    - **API 명세**: 사용자 생성(`POST /api/users`), 프로필 조회(`GET /api/users/{id}`) 등 핵심 API 3종의 설계를 완료했습니다.
    - **로직 구현**: 사용자 정보 등록 시 TDEE를 자동 계산하여 데이터베이스에 저장하고, 이를 조회하는 기능의 **백엔드 로직 구현을 완료**했습니다.

- **Shopping 도메인 (Mock 구현 완료)**
    - **API 명세**: 상품 검색(`GET /api/shopping/search`), 상품 추천(`GET /api/shopping/recommendations`) API 설계 및 기본 구현을 완료했습니다.
    - **Mock 구현**: 실제 외부 API 연동에 앞서, **Mock Client를 구현**하여 상위 서비스 및 컨트롤러 계층의 개발을 완료했습니다. 이를 통해 향후 실제 API 연동 시 하위 클라이언트 코드만 교체하면 되도록 설계했습니다.
