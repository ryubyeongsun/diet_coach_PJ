# 오늘 할 일 및 역할 분담 제안

`IMPLEMENTED_FEATURES.md`의 내용을 바탕으로, 오늘 두 분이 진행할 작업을 제안합니다. 한 분은 백엔드, 다른 한 분은 프론트엔드에 집중하여 독립적으로 개발할 수 있는 **'일일 체중 기록'** 기능을 구현하는 것을 추천합니다.

---

### **개발자 A (Backend Focus): '일일 체중 기록' API 개발**

- **목표**: 사용자가 매일 체중을 기록하고, 기간별 변화를 조회할 수 있는 백엔드 API를 구축합니다.

- **세부 작업 순서**:
    1.  **DB 설계**: `weight_records` 테이블을 `schema.sql`에 추가합니다.
        -   **컬럼 예시**: `id`, `user_id`, `weight`, `record_date`, `created_at` 등
    2.  **Domain/DTO 정의**: `WeightRecord.java` 엔티티와 API 요청/응답에 사용할 DTO(`WeightRecordRequest`, `WeightRecordResponse` 등)를 생성합니다.
    3.  **Mapper 구현**: MyBatis를 사용하여 `WeightRecordMapper` 인터페이스와 XML 쿼리(체중 삽입, 기간별 체중 조회)를 작성합니다.
    4.  **Service 구현**: `WeightRecordServiceImpl`에 체중을 저장하고 조회하는 비즈니스 로직을 구현합니다.
    5.  **Controller 구현**: `WeightRecordController`를 생성하고, 아래 두 개의 API 엔드포인트를 구현합니다.
        -   `POST /api/weights`: 오늘의 체중을 기록하는 API
        -   `GET /api/users/{userId}/weights`: 특정 기간 동안의 체중 기록을 조회하는 API (예: `?days=30`)

---

### **개발자 B (Frontend Focus): '체중 기록 및 시각화' UI 개발**

- **목표**: 사용자가 체중을 입력하고, 자신의 체중 변화를 차트로 한눈에 볼 수 있는 화면을 개발합니다.

- **세부 작업 순서**:
    1.  **체중 입력 UI 개발**: `MealPlanPage.vue` 또는 별도의 새 페이지에 오늘 체중을 입력할 수 있는 `input`과 '저장' 버튼을 포함한 컴포넌트를 추가합니다.
    2.  **체중 저장 API 연동**: '저장' 버튼 클릭 시, 개발자 A가 만들 `POST /api/weights` API를 호출하여 입력된 체중을 서버에 전송하는 로직을 구현합니다.
    3.  **체중 변화 차트 컴포넌트 개발**:
        -   개발자 A가 만들 `GET /api/users/{userId}/weights` API를 호출하여 체중 기록 데이터를 가져옵니다.
        -   `Chart.js`, `ECharts` 등 Vue용 차트 라이브러리를 설치합니다.
        -   가져온 데이터를 사용하여 체중 변화를 보여주는 라인 차트를 렌더링하는 컴포넌트를 만듭니다.
    4.  **통합**: 생성한 입력 UI와 차트 컴포넌트를 페이지에 배치합니다.

---

### 이 역할 분담의 장점

-   **독립적인 작업**: 백엔드 API 명세(URL, 요청/응답 형식)만 초기에 합의하면, 각자 다른 부분의 코드를 수정하므로 충돌(Conflict)이 발생할 확률이 거의 없습니다.
-   **병렬적인 개발**: 한 사람이 API를 만드는 동안 다른 사람은 UI를 만들 수 있어 프로젝트를 빠르게 진척시킬 수 있습니다.
-   **Full-Stack 경험**: 두 분 모두 새로운 기능을 DB부터 UI까지 완성하는 전체 흐름에 기여하게 됩니다.
